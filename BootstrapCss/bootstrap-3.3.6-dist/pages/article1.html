An article on MongoDb

by George Pottanat

MongoDB is one of the most widely talked about databases for Big Data Application. In today’s development environment, there are three types of database designs: RDBMS, OLAP and NoSQL. Developed by 10gen, MongoDB is a NoSQL database which means that it’s a non-relational database. “Scalable, open source, high performance, document oriented database” – 10gen Non-relational means that it doesn’t express the relational algebra like SQL. A non-relational database is a database where there are no joins, no complex transactions (multi-row) and can be a horizontally scalable architecture. Scalability is why many companies choose this Document Oriented Database. However some businesses don’t use MongoDB if the business needs a database which handles very complex transactions. In MongoDB the basic structure of the document oriented database: • database • collection (like a SQL table) • document (like an SQL row). When data is saved into MongoDB, it’s stored as a JSON store-structured documents expressed using flexible and light-weight JSON (JavaScript Object Notation). Ex. JSON document: { key1: ‘value1’, key2: ‘value2’, key3: [‘value3’, ‘value4’, ‘value5’] }; Unlike SQL with set tables and columns, MongoDB is great because data can have a Dynamic and Flexible schema. Suppose one document has a key4 and the other documents don’t have a key4, in SQL you can’t add the data of key 4 into a list in the same table as the other document with no key4. Within MongoDB database, the data is stored as BSON (sterilization format). This is how it creates flexibility to skip around in the document oriented database. Here is a list of companies who use MongoDB: • Walmart • Adobe • Craigslist • eBay • FIFA (EA Sports) • FourSquare • LinkedIn • McAfee • SAP • Shutterfly • FaceBook • AstraZeneca • Cisco • BuzFeed • Medtronic • eHarmony • Google • New York Time
